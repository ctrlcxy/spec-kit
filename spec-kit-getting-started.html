Spec Kit 项目零基础指南

这份说明专门为第一次接触 Spec Kit 的朋友准备，不需要任何编程经验。请从头到尾照做，每一步都包含要输入的命令和注意事项。

1. 项目是什么

Spec Kit 是 GitHub 打造的一套“先写清楚需求，再让 AI 帮你写代码”的工具。核心命令叫 Specify CLI，它可以：

  • 生成带有规范文档的项目骨架，让团队从需求开始协作。
  • 一次性准备好不同 AI 助手（Claude、Cursor、Copilot 等）的文件夹和命令。
  • 提供脚本，帮你把规范同步到各个 AI 助手的上下文里。
  • 打包模板，方便分享或上线。

记住： Spec Kit 是流程工具，本身不会自动写功能代码；它帮助你把“需求 → 计划 → 任务 → 实现”串起来。

2. 开始前准备

你的 Mac（macOS 15）上已经有 Git、Python、Node.js 和 VS Code，非常棒。我们还需要确认这些工具的位置，并准备一个终端窗口。

  1. 步驟 1：打开终端

    依次点击“启动台 → 其他 → 终端”。

    注意： 终端窗口里可以输入命令，回车后执行。所有命令都区分大小写。

  2. 步骤 2：确认 Python 版本

    python3 --version

    注意： 输出需要是 3.11 或更高（例如 3.12.1）。如果更低，请先运行 brew install python@3.11 并重新打开终端。

  3. 步骤 3：确认 Git 正常

    git --version

    注意： 只要出现版本号（例如 2.45.0），就说明可以继续。

  4. 步骤 4：安装或升级 uv（没有安装时执行）

    curl -LsSf https://astral.sh/uv/install.sh | sh

    注意： 这是官方脚本，会下载并自动安装 uv。执行后请关闭并重新打开终端，让配置生效。

  5. 步骤 5：把 uv 加入环境变量

    echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

    注意： 如果你使用的不是 zsh（macOS 默认是 zsh），需要替换成对应的配置文件，比如 ~/.bashrc。

3. 用最简单的方式体验 Spec Kit

接下来我们安装 Specify CLI 并创建一个示例项目。所有命令请按顺序输入。

  1. 步骤 1：安装 Specify CLI

    uv tool install specify-cli --from git+https://github.com/github/spec-kit.git

    注意： 需要联网。若网络不稳定，可多试几次；完全离线的做法在后面“从源代码运行”里。

  2. 步骤 2：检查命令是否可用

    specify --help

    注意： 如果显示“command not found”，说明 PATH 还没生效。回到上一节的步骤 5。

  3. 步骤 3：创建第一个 Spec Kit 项目

    specify init my-first-spec --ai claude

    注意： my-first-spec 是新项目的文件夹名称；--ai claude 可以改成 cursor、copilot 等。命令执行成功后，会在当前目录生成对应文件夹。

  4. 步骤 4：检查环境是否齐全

    cd my-first-spec
specify check

    注意： 终端切换到了新项目目录。检查结果会列出各种工具，✅ 表示检测通过，⚠️ 表示可选安装。

4. 推荐的工作流程（规范驱动）

Spec Kit 建议你让团队按照下面的顺序推进。命令以 “/” 开头，是在支持这些命令的 AI 助手（如 Claude CLI）里执行的。

  1. 步骤 1：写下团队原则

    /speckit.constitution 请生成一份确保代码质量、测试覆盖和用户体验一致性的团队原则

    注意： 说明你的红线，比如必须写测试、UI 风格要统一，这些会指导后面的所有工作。

  2. 步骤 2：描述要做的功能

    /speckit.specify 详细描述你想做的功能，重点写用户要完成什么事情

    注意： 写“用户点击按钮后会看到什么”，而不是“我要用 React 写一个按钮”。

  3. 步骤 3：补充技术方案

    /speckit.plan 说明你准备用什么技术栈、数据存储、性能要求以及安全考虑

    注意： 如果之后计划有变化，一定要返回来更新，否则 AI 可能按照旧方案继续。

  4. 步骤 4：拆解任务

    /speckit.tasks

    注意： AI 会给出多个任务。逐条过一遍，确认顺序和负责人，必要时补充说明。

  5. 步骤 5：着手实现

    /speckit.implement

    注意： 建议每完成一个任务就把成果和规范对比，如有偏差及时更新规格文档。

5. 如果想研究源代码或贡献

下面的步骤适合想要自行修改 Spec Kit 的用户。命令稍多，但每一步都很直接。

  1. 步骤 1：下载代码仓库

    cd ~
mkdir -p Projects
cd Projects
git clone https://github.com/github/spec-kit.git
cd spec-kit

    注意： 将仓库放在你有读写权限的目录里，例如 ~/Projects。

  2. 步骤 2：安装项目依赖

    uv sync

    注意： 该命令会自动创建 .venv 虚拟环境并安装所有依赖。如果看见“command not found”，说明 uv 没安装或 PATH 没配置好。

  3. 步骤 3：进入虚拟环境

    source .venv/bin/activate

    注意： 执行后，终端提示符前面会多出 (.venv)。完成工作后可输入 deactivate 退出。

  4. 步骤 4：以源码方式运行 CLI

    uv run specify --help

    注意： uv run 会确保使用当前目录的源码来运行命令，便于调试改动。

  5. 步骤 5：运行自动化测试（可选）

    uv run pytest

    注意： 在提交代码前执行。若测试失败，请阅读终端中的错误提示，通常是依赖问题或代码改动造成的。

  6. 步骤 6：生成模板压缩包

    bash .github/workflows/scripts/create-release-packages.sh

    注意： 运行前请确保安装了 zip 命令（可用 brew install zip）。脚本会在 .genreleases 目录下生成不同 AI 助手的模板。

6. 发布和分发的两种方式

6.1 当作 CLI 工具分发

  1. 步骤 1：构建安装包

    uv build

    注意： 会在 dist 目录生成 .whl 和 .tar.gz 文件。发布前要同步更新 pyproject.toml 的版本号以及 CHANGELOG.md。

  2. 步骤 2：本地安装测试

    uv tool install --from dist/specify_cli-<版本号>-py3-none-any.whl

    注意： 把命令里的 <版本号> 换成实际数字，确保安装包可正常使用。

  3. 步骤 3：上传到共享位置

    把 dist 目录中的文件上传到内网 PyPI、GitHub Release 或公司内部文件服务器。

    注意： 公开发布前请确认遵循 MIT 许可证的要求。

6.2 生成模板压缩包分享给团队

  1. 步骤 1：运行发布脚本

    bash .github/workflows/scripts/create-github-release.sh v1.2.3

    注意： 把 v1.2.3 替换成目标版本。脚本需要安装并登录 gh（GitHub CLI）。

  2. 步骤 2：查看输出文件

    脚本会在 .genreleases 目录生成多个 zip 包，对应不同的 AI 助手。

    注意： 如果只想生成文件、不想真正发布，可以在脚本运行后取消 gh release create 相关步骤。

  3. 步骤 3：分发给同事

    把相应的 zip 包给到队友，他们解压后就能得到包含 Spec Kit 结构的模板。

    注意： 即使使用离线模板，也推荐在新项目里运行 specify init 来补全设置。

7. 常见问题排查

  • 命令里出现 Permission denied： 确认仓库在你有写入权限的目录，例如 ~/Projects。
  • 报错找不到某个 CLI： 运行 specify check，根据提示的官网链接安装对应工具。
  • 无法访问网络安装失败： 可以在有网络的环境下载源码压缩包，拷贝到离线机器后运行 uv tool install --from /路径/spec-kit。
  • zip 或 gh 命令不存在： 使用 brew install zip gh 后重新执行相关脚本。
  • 遇到英文报错看不懂： 复制报错内容，翻译后查看关键字；大多数问题是路径、权限或版本不匹配。

8. 推荐阅读

  • 官方文档（含图文教程）

  • AGENTS.md：教你如何加入新的 AI 助手。
  • docs/ 目录：包含设计思路和示例。
  • GitHub Release 页面：查看每个版本的变更记录。

9. 下一步建议

  • 先按照“快速体验”章节完成一个示例项目，熟悉命令和目录结构。
  • 根据自己常用的 AI 助手，调整项目中生成的配置文件。
  • 阅读 spec-driven.md，尝试完整走一次“规范 → 计划 → 任务 → 实现”。
  • 如果准备贡献代码，请先阅读 CONTRIBUTING.md 并确保测试通过。
